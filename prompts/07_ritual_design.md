# 07 Ritual Design

## Purpose

Turn the life loop into a repeatable interaction.

## Input

- core_loop
- broken_points
- memory_contract
- sub_agents

## Output

- user_rituals:
  - name
  - trigger
  - cadence
  - duration
  - opening_move
  - steps
  - artifact
  - low_energy_version
  - success_signal

## Constraints

Design one core ritual for the agent.

Specify:

- name
- trigger
- cadence
- duration
- opening move
- 3 to 7 questions or actions
- closing artifact
- low-energy version
- success signal

The ritual should feel natural enough to repeat without ceremony.
Do not propose implementation, code, automations, database schemas, or full agent architecture at this stage.
