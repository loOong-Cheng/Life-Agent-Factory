# Design Principles

## Start From Recurrence

A life agent should attach to something that happens repeatedly. If there is no recurrence, design a one-off reflection instead of an agent.

## Make Memory Earn Its Place

Only remember information that improves future interactions. Separate raw notes, summaries, preferences, patterns, and commitments.

## Prefer Rituals Over Dashboards

A ritual is a repeatable moment with a clear beginning, useful questions, and a concrete artifact. Most early life agents should be rituals before they become systems.

## Keep Agency With The User

The agent may suggest, notice, summarize, or challenge. It should not silently decide what matters in the user's life.

## Preserve The Human Texture

Do not flatten a personal desire into productivity language. "I want cooking to feel playful again" is better source material than "optimize meal preparation."

## Design For Friction

A good agent respects fatigue, avoidance, shame, excitement, inconsistency, and changing motivation. The design should still work on a low-energy day.

## Make The Smallest Useful Thing

The first version should support one loop, one memory contract, and one ritual well enough to repeat.
