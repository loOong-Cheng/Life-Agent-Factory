# Life Agent Factory Method

Life Agent Factory is a method for designing personal agents from recurring life loops. A life loop is any repeated cycle of desire, action, record, interpretation, adjustment, and return.

The method is useful when the user says something like "I want to improve cooking," "I want to reflect on dancing practice," or "I need help thinking through research." These are not yet agent specs. They are invitations to discover what should be noticed, remembered, questioned, and repeated.

## Method Stages

1. Intake normalization: translate the user's vague desire into a working life area without narrowing too early.
2. Life scan: find recurring moments where the user already acts, avoids, records, forgets, compares, or reflects.
3. Domain diagnosis: identify the kind of life work involved.
4. Loop extraction: name the smallest repeatable loop worth supporting.
5. Memory contract: define what is captured, how it is summarized, and what requires consent.
6. Role decomposition: divide the agent into clear behaviors rather than one vague helper.
7. Ritual design: create a recurring interaction with triggers, prompts, outputs, and cadence.
8. MVP compression: remove everything not needed for the first useful loop.
9. Blueprint writing: produce a structured Agent Blueprint.
10. Review and revision: evaluate for usefulness, respect, specificity, and repeatability.

## Design Question

The factory's central question is:

What life loop is broken, worth recording, worth improving, or worth reflecting on?

Everything else follows from that.
